## 🔍 问题诊断

### 根本原因

**退出码 127**: "命令未找到"

**具体错误**:
```bash
./test_all_tools_cli.sh: line 242: timeout: command not found
```

### 问题分析

1. **平台兼容性问题**
   - `timeout` 命令是 GNU coreutils 的一部分
   - macOS 默认不包含此命令
   - 测试脚本在 Linux 上可以运行，但在 macOS 上失败

2. **影响范围**
   - 所有 96 个测试全部失败
   - 没有一个测试能够实际执行
   - 测试耗时都是 0 秒（立即失败）

3. **错误传播**
   - 脚本在第 242 行调用 `timeout 30s`
   - shell 无法找到 `timeout` 命令
   - 返回退出码 127
   - 测试被标记为失败

---

## ✅ 解决方案

### 修复方法

**替换 timeout 命令为 macOS 兼容的实现**:

```bash
# 原代码（不兼容 macOS）
timeout 30s cargo run --release --bin clawmaster -- agent --message "$input"

# 新代码（macOS 兼容）
cargo run --release --bin clawmaster -- agent --message "$input" &
local cmd_pid=$!

# 等待最多30秒
local timeout=30
local elapsed=0
while kill -0 $cmd_pid 2>/dev/null && [ $elapsed -lt $timeout ]; do
    sleep 1
    ((elapsed++))
done

# 检查进程是否还在运行
if kill -0 $cmd_pid 2>/dev/null; then
    # 超时，杀死进程
    kill -9 $cmd_pid 2>/dev/null
    wait $cmd_pid 2>/dev/null
else
    # 进程已完成
    wait $cmd_pid
    local exit_code=$?
fi
```

### 修复详情

**修改文件**: `test_all_tools_cli.sh`  
**修改行数**: 242-283  
**修改类型**: 超时机制重写

**新实现特点**:
1. ✅ 使用后台进程 (`&`)
2. ✅ 使用 `kill -0` 检查进程状态
3. ✅ 使用循环实现超时等待
4. ✅ 使用 `kill -9` 强制终止超时进程
5. ✅ 完全兼容 macOS 和 Linux
