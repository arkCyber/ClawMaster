### **场景 2: 开发者 - 代码审查**

#### **用户需求**
> "帮我审查这段代码，找出潜在问题"

#### **示例代码**
```rust
fn divide(a: i32, b: i32) -> i32 {
    a / b
}
```

#### **预期 AI 响应**
```
我发现了以下问题：

⚠️ **严重问题**
1. **除零错误风险**
   - 当 b = 0 时，程序会 panic
   - 这是未定义行为

💡 **改进建议**

方案 1: 使用 Result 类型
```rust
fn divide(a: i32, b: i32) -> Result<i32, String> {
    if b == 0 {
        Err("除数不能为零".to_string())
    } else {
        Ok(a / b)
    }
}
```

方案 2: 使用 Option 类型
```rust
fn divide(a: i32, b: i32) -> Option<i32> {
    if b == 0 {
        None
    } else {
        Some(a / b)
    }
}
```

推荐使用方案 1，因为它提供了更详细的错误信息。
```
