## 🔒 安全审计

### 安全问题清单

#### 高优先级 ⚠️

1. **沙箱隔离不足**
   - 当前实现仅使用 tokio::spawn_blocking
   - 缺少真正的进程隔离
   - 建议: 集成 nsjail 或 firejail

2. **插件签名验证缺失**
   - 无法验证插件来源
   - 存在恶意插件风险
   - 建议: 添加 GPG 签名验证

3. **输入验证不够严格**
   - 某些路径验证可能被绕过
   - 建议: 使用更严格的路径规范化

#### 中优先级 ⚠️

4. **配置文件权限**
   - 配置文件可能包含敏感信息
   - 建议: 设置适当的文件权限 (0600)

5. **依赖供应链安全**
   - 缺少依赖完整性检查
   - 建议: 使用 cargo-audit

6. **日志敏感信息泄露**
   - 可能在日志中记录敏感信息
   - 建议: 添加敏感信息过滤

#### 低优先级 ℹ️

7. **错误信息详细度**
   - 某些错误信息可能泄露系统信息
   - 建议: 区分开发和生产环境的错误信息
