## 🐛 问题描述

用户反馈：暗模式下界面仍然是**冷色调**（蓝灰色），而不是预期的**暖色调**（棕橙色）。

**截图显示**:
- 背景: 冷蓝灰色 (#0f1115)
- 文本: 冷白色 (#e4e4e7)
- 整体感觉: 冷色调 ❄️

**预期效果**:
- 背景: 暖棕色 (#1a1612)
- 文本: 米黄色 (#e8e3d8)
- 整体感觉: 暖色调 🔥

---

## 🔍 根本原因

### 问题 1: CSS 选择器错误

**错误代码** (`warm-dark-theme.css`):
```css
/* 错误：使用了 :root 选择器 */
:root,
[data-theme="dark"] {
  --bg: #1a1612;
  /* ... */
}
```

**问题**: 
- `:root` 选择器会影响**所有主题**（包括亮色模式）
- 导致暖色调变量被应用到所有场景
- 可能与其他 CSS 产生冲突

### 问题 2: CSS 加载顺序

虽然已经调整了加载顺序，但由于选择器问题，暖色调仍未正确应用。

---

## ✅ 修复方案

### 修复 1: 移除 :root 选择器

**修复后代码** (`warm-dark-theme.css`):
```css
/* 正确：只为暗模式定义暖色调 */
[data-theme="dark"] {
  --bg: #1a1612;  /* 深棕色背景 */
  --bg-elevated: #221e1a;
  --bg-hover: #2a2520;
  --surface: #1f1b17;
  --surface2: #252119;
  
  --text: #e8e3d8;  /* 暖白色文本 */
  --text-strong: #f5f0e5;
  --muted: #9a8f7e;
  
  --border: #3a342d;  /* 暖棕色边框 */
  --border-strong: #4a4238;
  
  --accent: #f59e0b;  /* 暖橙色强调 */
  --accent-hover: #d97706;
  --accent-dim: #b45309;
  --accent-subtle: #f59e0b1f;
  
  /* ... 其他变量 */
}
```

### 修复 2: 确保 CSS 加载顺序

**HTML 模板** (`index.html`):
```html
<link rel="stylesheet" href="{{ asset_prefix }}css/base.css">
<link rel="stylesheet" href="{{ asset_prefix }}css/warm-dark-theme.css">

```

**优先级**: `warm-dark-theme.css` > `base.css`

## 🔧 技术细节

### CSS 选择器优先级

1. **[data-theme="dark"]** - 特异性: (0,1,0)
   - 只在暗模式下生效
   - 不影响亮色模式

2. **:root** - 特异性: (0,0,1)
   - 影响整个文档
   - 会覆盖所有主题

**结论**: 使用 `[data-theme="dark"]` 更精确，避免副作用

### CSS 变量覆盖机制

```
加载顺序: base.css → warm-dark-theme.css → layout.css → ...

变量定义:
1. base.css: 定义 [data-theme="light"] 变量
2. warm-dark-theme.css: 定义 [data-theme="dark"] 变量
3. 其他 CSS: 使用 var(--bg) 等变量

结果: 暗模式使用暖色调，亮模式保持原样
```
