## 🏗️ 架构分析

### ClawMaster 工具架构

```
┌─────────────────────────────────────────┐
│           User / Client                 │
└─────────────┬───────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│         Gateway / Agent                 │
│  (智能决策哪些工具需要调用)              │
└─────────────┬───────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│        Tool Registry                    │
│  (注册和管理所有可用工具)                │
└─────────────┬───────────────────────────┘
              │
              ▼
┌─────────────────────────────────────────┐
│      Individual Tools                   │
│  (calc, exec, web_fetch, etc.)          │
└─────────────────────────────────────────┘
```

**关键设计原则**:
- ✅ **安全性**: 工具不直接暴露给 CLI
- ✅ **智能性**: Agent 决定何时调用哪些工具
- ✅ **可靠性**: 每个工具独立测试和验证
- ✅ **可维护性**: 清晰的分层架构
