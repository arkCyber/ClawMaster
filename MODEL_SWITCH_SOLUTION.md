# ✅ 模型切换解决方案

**问题**: llama backend 已被 Llama 3.2 1B 占用，无法切换到 Qwen  
**解决方案**: 重启 ClawMaster 并选择 Qwen 模型  

---

## 🔍 问题原因

### 技术细节
```
LlamaBackend::init() 只能调用一次
当前: Llama 3.2 1B 已占用 backend
尝试: 切换到 Qwen 14B
结果: ❌ "initializing llama backend" 失败
```

**根本原因**: llama.cpp 的 backend 是全局单例，不支持运行时切换模型

---

## ⚡ 立即解决方案

### 方案 A: 重启并选择 Qwen（推荐）

```bash
# 1. 停止当前进程
pkill -f clawmaster

# 2. 启动并等待模型选择
./target/debug/clawmaster

# 3. 在启动时选择 Qwen 2.5 Coder 14B
```

### 方案 B: 继续使用 Llama 3.2 1B（快速）

**优点**:
- ✅ 立即可用
- ✅ 足够测试新闻功能
- ✅ 响应快速

**操作**:
1. 关闭模型选择器
2. 直接测试新闻功能
