## 📊 最终配置

### `~/.config/clawmaster/clawmaster.toml`
```toml
[providers.ollama]
base_url = "http://localhost:11434/v1"  # 关键：必须包含 /v1
models = ["qwen3.5:9b"]

[chat]
default_model = "ollama::qwen3.5:9b"
```

### 启动和使用
```bash
# 1. 启动 Ollama（如果未运行）
ollama serve

# 2. 启动 ClawMaster 后端
./target/release/clawmaster gateway
# 输出: listening on https://localhost:XXXXX

# 3. 使用 CLI（替换为实际端口）
CLAWMASTER_GATEWAY_URL=https://localhost:XXXXX \
  ./target/release/clawmaster agent \
  --message "你的问题"
```

## 🎓 关键经验

### 1. Provider 选择
✅ **推荐 Ollama**:
- 模型管理简单
- 配置直观
- 自动处理模型加载

❌ **不推荐 local-llm**:
- 需要代码注册
- 配置复杂
- 维护困难

### 2. API 配置要点
```toml
base_url = "http://localhost:11434/v1"  # 必须包含 /v1
```

### 3. 会话管理
切换模型时必须清除会话数据库：
```bash
rm -rf ~/.clawmaster/*.db*
```

### 4. 调试技巧
- 检查后端日志: `grep "model=" backend.log`
- 验证工具调用: `grep "tool_calls_count=" backend.log`
- 提取端口: `grep "listening on" backend.log`
